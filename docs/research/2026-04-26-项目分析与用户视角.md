## 一、这是什么

**一句话**：面向独立开发者的外链提交自动化工具。一份产品配置，一条命令，把产品提交到 250+ 目录站。

**核心理念**：你有产品但没时间一个个填表。本工具 + Claude Code = 说一句话挂机跑，不被封、有记录、能复盘。

**适合的人**：
- 刚上线 SaaS / 工具站的 1 人开发者，想最低成本拿外链
- 小团队运营，需要系统化跑通 100+ 站点，留下凭证可复盘
- Claude Code / Agent 用户，想把外链当任务编排出去

**不适合**：只提交 1-2 个站（手动更快）；需要 PR / 内容营销；Windows 且不熟命令行。

## 二、架构速览

```
用户入口
  ├── claude（读 CLAUDE.md 自动驱动）         ← 推荐路径
  ├── node src/cli.js <命令>                  ← CLI 路径
  └── node src/batch-submit.js               ← 博客评论路径

CLI → submit.js（调度 + 预检 + 计时 + nextSteps）
    → sites/*.js（适配器层：generic / saashub / uneed / ...）
    → browser.js → bb.js（bb-browser 真实 Chrome 子进程）

数据层
  config.yaml          产品信息（私有，gitignored）
  targets.yaml         250+ 站点目录，给 Claude / 人读
  submissions.yaml     提交历史，自动写入（含 duration_ms）
  logs/global-history.json  博客评论去重记录
  screenshots/         提交截图，{hostname}-{date}.png 命名
```

**bb-browser 是核心**：通过启动你本地 Chrome 进行操作，不是无头浏览器，过 Cloudflare、OAuth、人机验证全靠它。playwright 在 v2.2 已移除。

## 三、安装

**前置条件**：Node.js 18+ + Chrome 浏览器

```bash
# 1. 克隆
git clone https://github.com/s87343472/backlink-pilot.git
cd backlink-pilot
npm install

# 2. 安装 bb-browser
npm install -g bb-browser
bb-browser --version    # 确认显示 0.10.x+
```

## 四、配置产品信息

```bash
cp config.example.yaml config.yaml
```

编辑 `config.yaml`，只需填这几项：

```yaml
product:
  name: "你的产品名"
  url: "https://your-product.com"
  description: "一句话描述（< 160 字符）"
  long_description: |
    2-3 句详细描述。说清楚功能和目标用户。
    例：帮助独立开发者自动提交外链，节省 20+ 小时手动操作。
  email: "your@email.com"
  categories: [developer-tools]   # ai / productivity / saas 等
  pricing: free                    # free | freemium | paid

browser:
  engine: bb                       # 固定用 bb，不要改
```

## 五、第一次使用

### 路径 A — Claude Code（推荐，最省力）

```bash
# 进入项目目录，打开 Claude Code
claude
```

然后直接说：

```
帮我把 [产品名] 提交到免费目录站
```

2. 检查 bb-browser 是否安装 → 没有则引导安装
3. 启动 Chrome → 逐个提交

**就这样。不需要读其他文档，不需要记命令。**

## 六、命令手册（v2.2.0 完整版）

### submit — 提交到目录站

```bash
# 提交到命名适配器
node src/cli.js submit saashub --engine bb

# 提交到任意 URL（通用适配器，不需要写代码）
node src/cli.js submit https://any-directory.com/submit --engine bb

# 干跑，不实际提交
node src/cli.js submit saashub --dry-run

# 机器可读输出（Claude agent 使用）
node src/cli.js submit saashub --json
```

成功时输出耗时；失败时输出错误码 + **nextSteps**（可直接复制执行）：

```
❌ Failed: 404 — submit page gone

  Next steps:
    Mark site as dead:
      node src/cli.js mark-dead futuretools --yes
```

## 七、批量提交工作流

### 用 Claude Code（推荐）

说一句话即可：

```
提交到所有免费站，从没提交过的开始
```

Claude 按 CLAUDE.md 里的「Batch Submission Playbook」自动：
1. 读 `status` 看已提交记录
2. 筛 `targets.yaml` 中 `auto: yes` 且非 dead / paid 的站
3. 逐站提交，间隔 60-180s 随机
4. 失败时根据 `nextSteps` 自动执行（mark-dead / mark-manual / 重启 Chrome）
5. 结束后输出汇总

---

### 纯 CLI 批量（手动节奏）

```bash
# 查记录，避免重复
node src/cli.js status

# 一站一站提交，自己控制节奏
node src/cli.js submit futuretools --engine bb
# 等 1-2 分钟...
node src/cli.js submit uneed --engine bb
# 等 1-2 分钟...
node src/cli.js submit startup88 --engine bb
```

---

### 博客评论批量（单独路径）

```bash
# 先准备资源文件
cp resources/backlink-resources.example.json resources/backlink-resources.json
# 编辑 resources/backlink-resources.json，填入目标博客 URL

# 干跑
node src/batch-submit.js --dry-run --limit 5

# 正式跑
node src/batch-submit.js --limit 10 --engine bb
```

博客评论使用 5 个虚拟人格轮换 + 20+ 评论模板 + 15-45s 随机间隔，降低检测风险。

## 八、提交节奏规则

1. **不重复提交**：每次提交前先 `status` 确认，工具内置去重但不保证
2. **控制频率**：不同站点间隔 1-3 分钟，每天 5-10 个站
3. **失败了先处理**：看 `nextSteps` → 标记 dead 或 manual → 继续下一个
4. **截图是凭证**：每次提交自动保存到 `screenshots/{hostname}-{date}.png`

## 九、常见错误处理

| 错误 | 原因 | 处理 |
|------|------|------|
| `bb-browser cannot connect to Chrome` | Chrome 未启动 | `bb-browser open about:blank` |
| `Chrome may be unresponsive` | Chrome 卡死 | `pkill -f "bb-browser" \|\| true && bb-browser open about:blank` |
| `404 — submit page gone` | 提交 URL 已失效 | `node src/cli.js mark-dead <site> --yes` |
| `Page redirected to login` | 站点需要账号 | `node src/cli.js mark-manual <site> --yes` |
| `No recognizable form fields` | 通用适配器解析失败 | `node src/cli.js scout <url> --deep` |
| `CAPTCHA detected` | 人机验证 | 彩色验证码自动处理；其他类型跳过 |

不确定怎么办时，直接在 Claude Code 里描述报错，Claude 知道所有排错步骤。

## 十、目标站点库说明

`targets.yaml` 收录 250+ 站点，结构：

```yaml
- name: Future Tools
  submit_url: https://www.futuretools.io/submit-a-tool
  type: form          # form | github | article | listing
  auto: yes           # yes = 可自动提交；manual = 需账号交互
  lang: en            # en | zh | multi
  notes: Popular AI tool directory
```

| 字段值 | 含义 | 数量 |
|--------|------|------|
| `auto: yes` | 可自动提交 | 180 |
| `status: dead` | 已失效，跳过 | 45 |
| `status: paid` | 需付费 | 1 |

**注意**：targets.yaml 设计为给 Claude 和人读，代码不直接消费。Claude 筛选后逐个调用 `submit`，这是 agent-first 设计。

## 十二、写新适配器（开发者）

```bash
# Step 1：侦察目标站字段
node src/cli.js scout https://newsite.com --deep

# Step 2：参考 src/sites/saashub.js 结构，创建 src/sites/newsite.js
```

适配器接口契约：

```js
export default {
  name: 'newsite',
  url: 'https://newsite.com/submit',
  auth: 'none',          // none | email-password | oauth
  captcha: 'none',       // none | color | turnstile
  engine: 'bb',

  async submit(product, config) {
    // product: { name, url, utm_url, description, long_description, email }
    // config: { _engine, browser, ... }
    return { url: '...', confirmation: '...' };
  },
};
```

## 十四、外链策略参考

**为什么要做外链**：Google 排名 = 其他站点链接到你 = 投票。票越多、来源越权威，排名越高。

### 渠道 ROI 排序

1. **GitHub awesome-lists** — 最高 ROI，永久，$0，每个 5 分钟。用 `awesome` 命令生成 Issue。
2. **免费目录站** — targets.yaml 收录 250+ 个，本工具主战场。
3. **博客评论** — website 字段留链接，用 `batch-submit.js` 批量自动化。

### 提交节奏

- 不同站点间隔 **1-3 分钟**
- 每天 **5-10 个站**
- **同一产品不重复提交同一站点**

### 避坑清单

| 站点 | 原因 |
|------|------|
| IndieHub | 发布要 $4.9，不是真免费 |
| OpenHunts | 免费排队 51 周 |
| toolify.ai | $99 |
| Product Hunt | 反爬，只能手动 |
