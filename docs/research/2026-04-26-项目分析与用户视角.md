## 一、项目定位

**一句话**：面向独立开发者的「外链提交自动化工具包」——一个 config，一条命令，把产品批量提交到 259 个目录站。

**关键事实**：
- Node 18+ ESM 项目，纯 CLI，无服务端、无数据库
- v2.1.0，作者 sagasu，MIT 协议，仓库 [s87343472/backlink-pilot](https://github.com/s87343472/backlink-pilot)
- 双引擎策略：**bb-browser 为主**（真实 Chrome 子进程，反检测能力强），playwright 为备（已被反爬封锁，事实退役）
- 设计哲学：**CLAUDE.md 即说明书**——目标用户用 `claude` 直接对话即可，不需要读文档

## 四、关键数据流

### 4.1 单站提交 `submit saashub --engine bb`

```
cli.js
  → loadConfig()                       [校验 product.* 必填]
  → submit.js:submit('saashub')
      → loadAdapter('saashub')         [动态 import sites/saashub.js]
      → 预检 HTTP（404/500/redirect→login）
      → withBrowser({engine:'bb'})     [拉起 bb-browser]
      → adapter.submit(product, config)
          → 登录 / 填表 / humanType / 截图 / 点击
      → recordSubmission()             [追加 submissions.yaml]
```

### 4.2 批量博客评论 `node src/batch-submit.js --limit N`

```
读 resources/backlink-resources.json
  → 优先级打分（blog_comment > profile，has_url > 无，no_captcha > captcha）
  → 过滤 logs/global-history.json 已提交项
  → 逐条 submitBlogComment（5 人格轮换，模板生成评论）
  → 写 logs/submissions-{DATE}.json + global-history.json
  → 每条间隔 15-45s 随机
```

### 4.3 站点适配器接口契约

所有 [src/sites/*.js](../../src/sites/) `default export` 同一形状对象：

```js
{
  name,
  url,
  auth,        // none | email-password | oauth
  captcha,     // none | color | turnstile
  engine?,     // bb | playwright（覆盖全局）
  submit(product, config) → { url, confirmation }
}
```

`generic.js` 是兜底——通过 bb-browser 的 a11y snapshot 自动识别 name/url/email/description 字段。

**结构示例**：

```yaml
- name: Future Tools
  submit_url: https://www.futuretools.io/submit-a-tool
  type: form
  auto: yes
  lang: en
  notes: Popular AI tool directory
```

**重要**：targets.yaml 当前**不被代码消费**——它是给用户/Claude 阅读的清单，由 Claude 按规则筛选后逐个调用 `submit`。这是 agent-first 设计的体现。

## 七、设计亮点

1. **bb.js 的 evalClickReal**：不靠 Playwright `.click()`，而是在浏览器内派发完整 `mousedown → mouseup → click` 序列——绕过 React/Vue 组件忽略合成事件的问题。这是反爬攻防的实战经验沉淀。
2. **CLAUDE.md 即 UI**：项目把 agent 当成一等公民——README 里"最快入口"是 `claude` 而不是 CLI。259 站点表设计成 YAML 而非 DB，就是为了让 Agent 能直接读筛。
3. **预检机制** [submit.js](../../src/submit.js)：提交前先 HEAD 一次 URL，避免拉起浏览器才发现 404，省时间 + 不留指纹。
4. **批量人格轮换** [batch-submit.js](../../src/batch-submit.js)：5 个虚拟身份 + 20+ 评论模板 + 15-45s 抖动——博客评论场景的反检测套路。

## 九、用户视角

### 9.1 典型用户画像

| 画像 | 描述 | 目标 | 痛点 |
|------|------|------|------|
| **A — Indie Hacker（主力）** | 1 人开发者，刚上线 SaaS / 工具站，没 SEO 预算 | 让产品在 Google 上能被搜到，最低成本拿外链 | 不想花几小时手动一个个站填表；不懂哪些站值得提交；怕被封号 |
| **B — 中小团队 marketer** | 2-5 人小团队的运营，负责 SEO 冷启动 | 系统化跑通 100+ 站点提交，留下凭证可复盘 | 重复劳动；提交记录散落 Excel；漏交 / 重交 |
| **C — Agent 玩家** | 熟 Claude Code / OpenClaw，喜欢「说一句话就跑」 | 把外链当成 agent 任务编排出去 | 工具不是 agent-friendly（要读长文档、参数复杂） |
| **D — 二开开发者** | 想给自己的站做适配器、贡献给项目 | 快速理解适配器接口，复用 bb-browser 能力 | 缺 adapter 模板示例；不懂 bb-browser 反检测细节 |

仲长属于 **A + C 混合**——产品出身、用 Claude Code 做开发，是这个项目的核心目标用户。

### 9.2 用户旅程

#### 旅程一：Claude Code 用户（推荐路径，最短）

```
[发现] GitHub 看到 README → 注意到「don't need to read any docs」
   ↓
[安装] git clone + npm install + bb-browser → 30 秒
   ↓
[启动] 终端里 cd 项目 → 输入 `claude`
   ↓
[对话] 「帮我提交外链」
   ↓ Claude 读 CLAUDE.md → 触发 First-time User Flow
[配置] Claude 问 5 个问题（产品名、URL、描述、长描述、邮箱）→ 自动生成 config.yaml
   ↓
[启动浏览器] Claude 跑 `bb-browser open about:blank`
   ↓
[首次试水] Claude 跑 `submit futuretools --engine bb` 验证链路
   ↓
[批量] 用户说「提交到所有免费站」→ Claude 按 targets.yaml 筛选 → 1-3 分钟一站逐个提交
   ↓
[复盘] `node src/cli.js status` 查看提交历史 + screenshots/ 看截图
```

**触点体验关键节点**：
- ✅ 「3 步上手」承诺兑现：clone / install / claude
- ✅ Claude 主动问问题，用户不需要查文档字段
- ⚠️ bb-browser 必须先 `open about:blank` 启动 Chrome——这一步对非技术用户仍是黑盒
- ⚠️ Cloudflare Turnstile 需要 2 分钟内完成，慢站会失败
- ⚠️ 5 个问题问完才生成 config，中间不能保存草稿——重启对话要重问

#### 旅程二：纯 CLI 用户（无 agent）

```
[发现] README 看到 Manual Quick Start
   ↓
[配置] cp config.example.yaml config.yaml + 手动编辑（要懂 YAML）
   ↓
[运行] node src/cli.js submit futuretools --engine bb
   ↓
[扩展] 想批量？需要自己写 shell 循环或读 targets.yaml 脚本
```

**摩擦点**：CLI 没有「auto-batch」命令——`submit` 只能单站，要批量得用户自己拼命令或写脚本。`batch-submit.js` 是另一条独立路径（博客评论），跟目录站提交不通用。

#### 旅程三：二开开发者

```
[发现] 看到自己常用的某站没适配器
   ↓
[侦察] node src/cli.js scout https://newsite.com --deep → 看字段
   ↓
[尝试通用] node src/cli.js submit https://newsite.com --engine bb
   ↓ 通用适配器失败（字段名特殊 / 多步表单）
[写适配器] 参考 src/sites/saashub.js → 写 newsite.js
   ↓
[测试] 跑一次提交 → 看 screenshots/ 验证
   ↓
[贡献] PR 到上游
```

**摩擦点**：[docs/contributing.md](../contributing.md) 有规范但缺**最小可运行模板**；新人要对着 saashub.js 这种已有复杂逻辑的文件去临摹。

### 9.3 触点与体验热力图

| 触点 | 体验 | 备注 |
|------|------|------|
| README 首屏 | ⭐⭐⭐⭐⭐ | 「don't need to read any docs」承诺到位 |
| `claude` 对话引导 | ⭐⭐⭐⭐⭐ | CLAUDE.md First-time Flow 设计精良 |
| config.yaml 生成 | ⭐⭐⭐⭐ | 自动化好，但中断恢复差 |
| bb-browser 启动 | ⭐⭐⭐ | 黑盒 + 偶发卡死，需 `pkill` 救场 |
| 单站提交 | ⭐⭐⭐⭐ | 预检 + 截图 + 历史，闭环完整 |
| 批量提交（目录站） | ⭐⭐⭐ | **缺批量命令**，要么靠 Claude 循环、要么手动 |
| 批量提交（博客评论） | ⭐⭐⭐⭐ | batch-submit.js 体验好，但跟目录站两条路 |
| 状态复盘 | ⭐⭐⭐ | YAML + JSON 两份历史，要分别看 |
| 死站 / 失败处理 | ⭐⭐ | 失败后无明显「下一步」引导 |
| 二开适配器 | ⭐⭐⭐ | 文档全，但缺脚手架 / 模板 |

### 9.4 用户痛点 Top 5

1. **批量目录站提交无原生命令**——只能靠 Claude 循环 `submit`，纯 CLI 用户得自己写脚本。
2. **bb-browser 启动是黑盒**——卡死后用户不知道 `pkill -f bb-browser`，要回去翻 CLAUDE.md。
3. **历史数据双轨**——submissions.yaml（单站）和 logs/global-history.json（批量评论）分开存，跨命令查重失效。
4. **失败提示无下一步**——`404` / `login required` 等错误只是报告，不引导用户做什么（标记 dead？换 URL？转手动？）。
5. **适配器开发缺脚手架**——没有 `cli.js scaffold-adapter <name>` 这种命令快速生成模板。

### 9.5 价值主张（用户语言）

> 「我有产品，但没时间一个个去填外链表单。给我一个工具，让我说一句话就能把产品提交到几百个站，不被封、有记录、能复盘——这就是 Backlink Pilot。」

**核心价值**：
- **省时间**：259 站手动提交 = 20+ 小时；本工具 + Claude = 几小时挂机
- **低门槛**：不需要懂 SEO、不需要懂代码、不需要读文档
- **不被封**：bb-browser 真实 Chrome + 人格轮换 + 抖动延时 = 反检测内功
- **可复盘**：所有提交有 YAML 记录 + 截图凭证

**不适合的人**：
- 想提交 1-2 个站：手动更快
- 想做 PR / 内容营销：本工具不是这条路
- 用 Windows 且不熟命令行：bb-browser 体验受限

## 十、改进建议（优先级排序）

| 优先级 | 改进点 | 预期收益 |
|--------|--------|----------|
| **P0** | 加 `cli.js batch-submit --filter "auto:yes,status!:dead"` 命令 | 解决 Top 1 痛点，纯 CLI 用户可批量 |
| **P0** | 统一 submissions.yaml 与 logs/global-history.json | 解决 Top 3 痛点，跨命令去重 |
| **P1** | 失败时输出「下一步建议」（如 `node cli.js mark-dead <site>`） | 解决 Top 4 痛点 |
| **P1** | 添加 `scaffold-adapter` 命令生成模板 | 解决 Top 5 痛点，降低贡献门槛 |
| **P2** | bb-browser 健康检查命令 + 卡死自动恢复 | 解决 Top 2 痛点 |
| **P2** | 自动化清理 dead 站（每月跑一次健康检查） | 减少 Claude 重读判断负担 |
| **P3** | 剥离 playwright 依赖（事实退役） | 包体积 -45MB |
| **P3** | 加 `.claude-local.md` 自动初始化 hook | 兑现 CLAUDE.md 的承诺 |
