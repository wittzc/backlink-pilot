# nofollow 绕过实测（addfamilyphoto，2026-06-12）

> 一句话结论：**机制成立，端到端不可行。** WordPress 评论的 nofollow 绕过在引擎层、提交层都跑通了，但所有评论都卡在「博主审核 / Akismet spam」这一关出不来——没公开 = 没外链。真正的瓶颈不是 nofollow 绕过技术，是「评论能不能被公开」，而那由人 + 反垃圾系统决定，技术绕不过。
>
> 关联：方法论全景[第四部分](外链建设方法论全景.md)（绕过技术原文）、[METHODOLOGY](../METHODOLOGY.md) fail-fast 边界。

## 背景

方法论全景帖 05 记录了一个 WordPress 评论绕过：在评论正文的 `<a href="...\n">` 里塞真实换行，使 `wp_rel_nofollow()` 的正则 `|<a (.+?)>|i`（无 `/s` 标志，`.` 不跨行）漏匹配该标签 → 不加 nofollow → 链接保持 dofollow。本次用 addfamilyphoto.com 实测它「当前是否还有效 + 能不能产出真实外链」。

## 三层结论（逐层验证）

| 关 | 验证方式 | 结果 |
|---|---|---|
| **关 2：nofollow 正则绕过** | 隔离探针：用 `bb.js` 的 `evalFill` 把带真实换行的 `<a>` 送进 textarea，回读 `value` + 跑 WP 真实正则 | ✅ **成立**。换行字符（0x0A）完整保留进 `value`；`/<a (.+?)>/i` 对它 false（漏匹配 → 不加 nofollow），带 `/s` 的对照版 true。差异精确锁定在缺失的 `/s`。WordPress core 7.0 / trunk 该正则至今未修。 |
| **关 1：评论提交入库** | 真实提交到标准 WP 博客 | ✅ **成立**。lisalisson 第二次提交返回 WP core 的 `Duplicate comment detected` 错误页——只有第一次已入库才会触发，铁证评论 POST 成功进库。 |
| **收录关：评论公开** | 提交后重访文章页验 rel | ❌ **失败**。入库的评论全部 `found:0`、无 `awaiting moderation` 提示 → 进了人工审核队列或被 Akismet 判 spam，**无一公开**。 |

**方法论铁律**：「只有被公开的评论才是有效外链。」收录关失败 → 实际外链产出 = **0**。

## 实测数据（11 个站）

**首批 3 个标准自托管 WP 博客**（工具测评 / 家族史 niche，对口）：

| 站 | 入库 | 公开 | dofollow |
|---|---|---|---|
| thegenealogyreporter.com | ✅（提交后重定向首页） | ❌ | 验不到 |
| lisalouisecooke.com | ✅（重载文章页） | ❌ | 验不到 |
| lisalisson.com | ✅（duplicate detection 铁证） | ❌ | 验不到 |

**第二批 8 个站**：5 个 `no_comment_field`（非标准 WP 评论——Squarespace / JS / Disqus 结构，原生 WP 选择器匹配不到）、1 个无提交按钮、1 个 `ERROR 403`（Cloudflare/反爬）、1 个提交未触发（legacytree，截图显示表单原样）。**真正入库 0 个。**

合计：尝试 11 站 → 标准 WP 入库 3 个 → 公开 0 个 → dofollow 0 个。

## 为什么卡住（根因）

1. **成熟博客普遍装 Akismet + 人工审核**。评论含商业链接（addfamilyphoto）+ `evalFill` 瞬间填充（机器人行为指纹，非逐字键盘）→ 正是 Akismet 最爱拦的特征 → 静默进 spam，作者都看不到。
2. **关 1 的「反垃圾」维度 `evalFill` 过不去**。早先分析已指出：绕过技术要求 `pressSequentially` 逐字键盘输入来过反垃圾时序检测，而 `bb.js` 的输入路径（`type → fill → evalFill`）全是 set-value，没有键盘事件层。补它要改 bb 底层、且依赖 bb-browser 是否暴露键盘原语（未确认）。
3. 即便过了关 1，**收录关仍由博主人工审核把关**——技术无法绕过。

## 对产品的决策

**不把 nofollow 绕过纳入产品**（含独立 bypass 子分支）。这次是**实测否决**，比之前的 ROI 推断更硬：

- 收益分子被「审核/spam 拦截」压到实测 0，不只是「Google 可能不算权重」的理论 0；
- 成本不变：窗口期手法 + 需补 bb 键盘原语 + 每站对抗 Akismet；
- 与产品 verdict 收敛内核反向（灰色窗口期博概率 vs 可累积可复制）。

实验脚本 `src/experiments/nofollow-live-probe.mjs` 保留作复现入口（**实验性，会真实发评论，勿误运行**）；证据在 `reports/nofollow-probe-*.json` + `screenshots/nf-*.png`、`nofollow-*.png`。

## 真实发布影响（如实记录）

实测向 ~3-4 个真实博客留下了待审/spam 评论（署 persona 假名 + 假邮箱 + 带 addfamilyphoto 链接），均未公开（被审核/spam 拦）。首批 3 个标准 WP 站的评论进了审核队列/spam，博主可能在后台看到待审项。实际可见影响有限（无一公开）。
