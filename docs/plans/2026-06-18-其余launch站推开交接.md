## 前置条件（接手前先确认）

1. **登录态**：用户已用专用 Google 账号注册了 ~19 个 launch 平台。但 bb-browser 的 Chrome
   profile 不一定持有每个站的登录态——**每站动手前先 `bb-browser snapshot <dashboard-url>`
   确认是否登录**；掉登录就停下来让用户在 bb-browser 的 Chrome 里手动登一次（登录态会复用）。
2. **bb-browser 环境**：`bb-browser status`。若 `503 Chrome not connected` →
   `bb-browser daemon shutdown && bb-browser open about:blank`。版本 0.11.3，CDP 在 `127.0.0.1:19825`。
3. **不动代理**：代理/VPN 是高危，连不上就停下报告用户，不要自己改。

## 已验证的方法论 + 坑清单（来自 TinyLaunch）

逐站填表前默认这些坑都可能出现：

| 坑 | 现象 | 解法 |
|---|---|---|
| **React 受控组件** | bb-browser `fill`/`type` 改了 DOM `.value` 但提交发空值 | 用原生 setter + dispatch：`Object.getOwnPropertyDescriptor(HTMLInputElement.prototype,'value').set.call(el,v); el.dispatchEvent(new Event('input',{bubbles:true}))` |
| **文件上传（logo）** | bb-browser 无 upload 命令，`eval` 不能设 `input.files` | 原生 CDP `DOM.setFileInputFiles`（node22 内置 WebSocket 直连 CDP，无依赖；见 submit-tinylaunch.js） |
| **富文本编辑器** | contenteditable / ProseMirror，setter 无效 | focus + `execCommand('selectAll')` 后 CDP `Input.insertText`（selectAll 保证重跑幂等，否则追加重复） |
| **唯一性字段冲突** | 保存"静默回空"、解锁不了 | 拦截保存请求看后端真实报错（fetch override 捕获 response body），别靠猜 |
| **SPA 异步渲染** | 表单还没出来就填 → 元素 null 报错 | 填字段前 poll 关键 selector 出现再动手 |
| **Playwright connectOverCDP** | 和 bb-browser daemon 抢 CDP，30s 超时 | 不用 Playwright，用原生 CDP over WebSocket |

**判断基线（ROI，别忘）**：半自动只对**标准表单 + 免费 dofollow**的站值得。
React 多步 / captcha / 付费才 dofollow / 强人工审核 → 直接判手动或跳过，不硬啃
（对应 METHODOLOGY 原则 5 fail-fast）。

## Per-site 执行 protocol（每站重复）

1. **确认登录态**：`bb-browser snapshot <站的 dashboard/submit url>`，看是否已登录。掉登录 → 停，让用户登。
2. **scout 表单结构**：`bb-browser snapshot` + `bb-browser eval` dump 所有 `input`(type/name/placeholder)、
   `select` options、contenteditable、file input。判断字段类型。
3. **验 rel**（dofollow 不确定的站）：发一条后看产出链接的 `rel` 属性，nofollow 则降价值。
4. **判定**：
   - 纯标准表单（text/select，无必填文件上传、无 captcha、无多步）→ **写 recipe** `recipes/<site>.yaml`
     （参考 recipes/aivalley.yaml 格式：selector→value 映射 + submit selector）。
   - 有文件上传 / 富文本 / React 多步但仍单页 → **仿 submit-tinylaunch.js 写专用 CDP 脚本**
     （复用上面坑清单的解法）。
   - captcha / 付费才 dofollow / 强人工审核多步 → **判手动**，targets 标注原因。
5. **真发**：填好先 dry-run 截图自检（name/url/category/logo 对不对），**再提交**。
   ⚠️ 提交是对外发布，不可逆——发布前用截图确认，重要的站可以让用户过目。
6. **留档**（每站完成后立即）：
   - `targets.yaml` 该站 notes 更新（auto 状态、脚本/recipe 路径、发布结果、坑）。
   - 研究笔记追加该站的 scout 发现 + 判定。
   - 全部推完后更新 `docs/context.md` 的 Next。
