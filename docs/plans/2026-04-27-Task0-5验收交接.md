## 1. 任务 A：Browser triage 复跑（criterion 3）

### 命令

```bash
# 备份旧报告（避免覆盖丢失基线）
mv reports/triage-browser-overseas-ai-top30.json \
   reports/triage-browser-overseas-ai-top30.v1.json 2>/dev/null || true

# 跑新版
node src/cli.js triage --browser --category overseas_ai_directories --limit 30 \
  --json --output reports/triage-browser-overseas-ai-top30.json
```

预计耗时 **5-10 分钟**（30 站串行，每站 5-15 秒，含 bb-browser DOM 抓取）。

### 通过判据

跑完后用下面命令检查：

```bash
node -e "
const j = require('./reports/triage-browser-overseas-ai-top30.json');
console.log('total:', j.summary.total);
console.log('buckets:', JSON.stringify(j.summary.buckets));
console.log('manual_reasons:', JSON.stringify(j.summary.manual_reasons));
console.log('tiers:', JSON.stringify(j.summary.tiers));
console.log('bucket_by_tier:', JSON.stringify(j.summary.bucket_by_tier));
console.log('value_tier present on all:', j.results.every(r => 'value_tier' in r));
console.log('reason on manual:', j.results.filter(r=>r.bucket==='manual-review').length, '条 manual-review');
console.log('  其中带 reason:', j.results.filter(r=>r.bucket==='manual-review' && r.reason).length);
"
```

**判 ✅** 若同时满足：
- `total` 在 **25-30** 范围（部分站点可能 dead/timeout，正常）
- `summary.buckets` 是非空对象，**至少出现** `provider-ready` 或 `recipe-ready` 或 `manual-review` 之一
- `summary.bucket_by_tier` 是二维对象（bucket → tier → count）
- `summary.manual_reasons` 是非空对象（至少 `captcha-required` / `login-required` / `paid` / `unknown` 之一）
- `value_tier present on all` 是 **true**
- 所有 `manual-review` 条目都带 `reason` 字段（"其中带 reason" 等于 "manual-review 总数"）

**判 ⚠️** 若 total 在 1-24 之间（部分跑通但有大量 timeout）→ 接受为部分通过，记录 timeout 站点数。

**判 ❌** 若崩溃、文件没生成、或上面任意 ✅ 项不满足。

### 失败处理

- Chrome 死了 → `pkill -f "bb-browser" || true && bb-browser open about:blank`，重跑一次
- 个别站 timeout → **正常**，被分类为 `dead` 或 `unknown`，不算失败
- 整命令崩溃 → 把 `stderr` paste 到报告，**不要改代码**
- 跑超过 15 分钟 → ctrl+C，记录已完成多少站

## 2. 任务 B：Recipe smoke test（criterion 5）

### 2.1 AI Valley（无 captcha，应该全自动通过）

```bash
node scripts/recipe-smoke-test.js aivalley 2>&1 | tee /tmp/smoke-aivalley.log
```

预计耗时 **30-90 秒**。

期望最后输出 JSON 类似：
```json
{
  "siteKey": "aivalley",
  "url": "https://aivalley.ai/contact/",
  "dryRun": true,
  "readback": {
    "name": "<config.product.name 的值>",
    "email": "<config.product.email 的值>",
    "ToolName": "<同 product.name>",
    "ToolURL": "<config.product.url 或 utmUrl>",
    ...
  }
}
```

**判 ✅** 若同时满足：
- 退出 code = 0
- stdout 含 JSON 输出
- `dryRun: true` 字段存在
- `readback` 至少有这 4 个字段且值与 config 一致：name / email / ToolName(=name) / ToolURL(=url 或带 UTM 的)
- **没有任何 "submit clicked" 或类似输出**（脚本三层 dryRun 防护应阻止）

### 2.2 Future Tools（Cloudflare Turnstile，半自动）

```bash
node scripts/recipe-smoke-test.js futuretools 2>&1 | tee /tmp/smoke-futuretools.log
```

预计耗时 **30-90 秒**。

**两种合理结果都判 ✅：**

A. **Smoke script 主动 abort 因检测到 Turnstile**
   - stderr 含类似 `CAPTCHA detected` / `Turnstile` / `aborting smoke test`
   - **这是预期行为**（ADR-007 captcha fail-fast）

B. **Smoke script 完成填表 + readback**
   - 输出 JSON，`readback` 含 name / email / tool_name / tool_url / description 至少 5 项匹配 config
   - Turnstile 没被解（agent 不绕过 captcha）但 fill 路径正常

**判 ❌** 仅当：
- 脚本崩溃（非主动 abort）
- 脚本声称点了 submit（stderr 含 "submit clicked" 或 "submitted: true"）

### 失败处理

- `config.yaml` 报错 → 按 0.3 重新建
- bb-browser 连不上 → 0.2 重启 Chrome
- 任何 "DRY_RUN check failed" 类错误 → **停**，paste 到报告，**不要 force**（这是安全 bug，需要原 agent 看）
