# Backlink Pilot 跨 Agent 通用化协议 Implementation Plan

**Goal:** 让 backlink-pilot 的 AI 交互层不绑定任一 agent 工具——任何 agent（Claude Code / Codex / Cursor / Copilot / Aider 等）打开项目都能找到自己的入口、读到一致的指令、用同一份命令清单。

**Architecture:** 单一权威源 + tool-specific redirect stub + machine-readable manifest。

- `docs/AGENT_GUIDE.md` —— 唯一权威源，存所有 onboard 流程、命令说明、错误处理、playbook
- 各工具入口（`CLAUDE.md` / `AGENTS.md` / `.cursorrules` / `.github/copilot-instructions.md`）—— 一屏 stub 指向 AGENT_GUIDE
- `agent-manifest.json` —— 结构化命令目录，让 agent 不读 markdown 也能调度命令

**Tech Stack:** plain markdown + JSON。无运行时依赖。

**验证方式（必填，必须可观测）:**
- 4 个 stub 文件存在且都引用 `docs/AGENT_GUIDE.md`
- AGENT_GUIDE.md 顶部含明确「single source of truth」声明 + exit code 约定段
- agent-manifest.json 通过 `JSON.parse` 校验
- manifest 含 ≥ 6 个核心命令的元数据（包含 syntax / purpose / json 字段）
- 旧 AGENTS.md 的 codex-fork 内容已被覆盖为 stub（不再有过期的"v2.1 — Codex Instructions"）

## Plan Formation and Decision Rationale

- **Problem statement:**
  - 用户希望项目能被任何 agent 工具调用，不依赖单一 AI（"我希望是不同的 Agent 工具都能支持调用，不依赖于某一个"）
  - 现状：CLAUDE.md 是写给 Claude Code 的；AGENTS.md 是写给 Codex 的（且已 fork-drift 为旧版本 v2.1）；其他 agent 工具进来无入口
  - 直接后果：双份维护必然漂移（实测 AGENTS.md 已经把 `.claude-local.md` 错改成 `.Codex-local.md` 这种细节，证明同步滞后）

- **Options considered:**
  - A. 每个 agent 工具一份独立 instruction 文件（现状的扩展版）— 否定（N 份必然漂移，已经发生过）
  - B. 完全靠 README 自描述 — 否定（用户的 200 行 know-how 会丢失，agent 起步阶段慢）
  - C. 单一权威源 + tool-specific stub + machine-readable manifest — 采用

- **Final decision:** C

- **Why this option:**
  - 单源是消除漂移的根本方法（commit message 里也写明：N 独立 copy 必然 drift）
  - stub 模式让"加新 agent 工具"成本为零（一个 stub 文件，3 行）
  - JSON manifest 提供程序友好的接入路径——bash 脚本、CI、外部 wrapper 都能消费，不只是 AI
  - 不破坏现有工具的发现机制——每个工具仍按自己的约定找入口文件，只是入口内容是 stub

## 决策推导

**评估过的方案**：
- A. 每个 agent 工具一份独立 instruction 文件 — 否定（双份漂移现状已证明，AGENTS.md 已 fork drift）
- B. 完全靠 README + CLI --help 自描述 — 否定（200 行 onboard know-how 会丢，新 agent 起步成本高）

**采用方案**：单一权威源（docs/AGENT_GUIDE.md）+ tool-specific redirect stub（4 个）+ machine-readable manifest（agent-manifest.json）

**未来参考**：
- 加新 agent 工具支持时：只写一个 stub，不动权威源
- 改 agent 行为时：只动 docs/AGENT_GUIDE.md，stub 永远不动
- 程序化调度（bash / CI / external wrapper）：消费 agent-manifest.json，不解析 markdown
- 同样设计可复用到其他项目——这是项目无关的"AI 接入层"模式
