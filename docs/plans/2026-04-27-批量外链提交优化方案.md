**Goal:** 把 backlink-pilot 从「逐站尝试提交」升级为「先分桶、再批量、再按价值写 adapter」的批量外链提交系统。

**Architecture:** 采用三层策略：`triage` 先对 `targets.yaml` 中目标站点分桶，`generic` 处理可见简单表单，site-specific/provider adapters 处理高价值复杂表单和 iframe 表单。批量执行不再盲跑 180 个站点，而是基于分桶结果决定自动提交、专用适配器、人工复核或剔除。

**Tech Stack:** Node.js ESM、bb-browser、YAML targets、Node test runner、CLI JSON 输出。

**验证方式（必填，必须可观测）:**
- 运行 `node src/cli.js triage --limit 20 --browser --json`，能看到 `generic-ready`、`adapter-needed`、`iframe-provider`、`manual-review` 等 bucket。
- 运行 `node src/cli.js triage --output reports/triage-http.json --json`，磁盘生成 JSON 报告，且包含 `summary.total` 与 `results[]`。

## Plan Formation and Decision Rationale

- Problem statement:
  - 用户目标是对 `targets.yaml` 中约 180 个 `auto: yes` 目标批量提交产品外链。
  - 现有 generic adapter 只能覆盖传统 HTML 或少量可见简单表单，面对 React 动态渲染、iframe 表单、验证码、登录墙、付费/关闭站点时失败原因混杂。
  - 如果直接批量提交，时间会被消耗在不可自动化页面上，也无法形成后续 adapter 排期。
- Options considered:
  - 继续扩大 generic adapter，一次性尝试覆盖所有站点。
  - 为所有站点手写 site-specific adapter。
  - 先做 triage 分桶，再对 `generic-ready` 自动提交，对高价值 `adapter-needed` 和 `iframe-provider` 写 adapter。
- Final decision:
  - 采用「triage 分桶 + generic 自动提交 + provider/site-specific adapters」的混合路线。
- Why this option:
  - generic 的边际收益集中在可见表单，无法可靠跨越 iframe、验证码、登录/付费。
  - 手写 180 个 adapter 成本过高，且大量站点已经不可用或价值低。
  - 分桶报告能把下一步工作转化为可排序的工程队列：先拿高价值站点和可自动站点的收益。

## Target Workflow

1. 快速粗筛:
   ```bash
   node src/cli.js triage --json --output reports/triage-http.json
   ```
2. 高置信浏览器分桶:
   ```bash
   node src/cli.js triage --browser --json --output reports/triage-browser.json
   ```
3. 提交策略:
   - `generic-ready`: 用 generic adapter 小批量提交，每批 5-10 个，间隔 60-180 秒。
   - `adapter-needed`: 按站点价值排序，优先 Future Tools、AI Valley、Futurepedia、Uneed、Toolify 等。
   - `iframe-provider`: 先做 Paperform/Tally/Typeform/Airtable provider adapters，再复用到多个站点。
   - `manual-review`: 登录、验证码、付费、关闭、网络不稳定，人工复核或标记 manual/dead/paid。
4. 结果闭环:
   - 成功提交写入 `submissions.yaml`。
   - 失败根据 `code` 触发 `mark-dead`、`mark-manual` 或 adapter backlog。
   - 每轮批量后输出成功/失败/跳过汇总。

### Task 4: 批量提交执行规则

**Batch policy:**
- 每批 5-10 个站点，避免浏览器状态污染和反自动化风险。
- 每站结束后记录 `submissions.yaml`。
- 失败立即分类，不在同一站上连续重试超过 2 次。
- 遇到 CAPTCHA/登录/付费直接转 manual，不阻塞批次。

**Verification:**
```bash
node src/cli.js status --json
node src/cli.js stats --timing --json
```
